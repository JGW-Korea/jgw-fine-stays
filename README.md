## 🏨 JGW Fine Stay

JGW Fine Stay는 바카티오에서 상용화 중인 [**FineStay**](https://finestays.com/search)를 클론 코딩한 프로젝트입니다.

<br />

## 🤔 기획 배경

Fine Stay를 대상으로 클론 코딩을 진행하게 된 이유는 **기술적 난이도가 높고**, **실제 서비스에서 발생할 수 있는 다양한 문제를 마주하고 이를 개선하는 경험**을 할 수 있다고 판단했기 때문입니다.

<br />

1. 모든 디바이스에 대응할 수 있도록 반응형으로 설계되었지만, **초기 렌더링 시 모든 정적 자원을 한 번에 요청하는 구조**로 인해 Lighthouse 측정 기준 데스크탑 환경에서는 **FCP 1.2초, LCP 23.4초**, 모바일 환경에서는 **FCP 2.9초, LCP 74.9초**로 측정되어 **사용자 입장에서 초기 로딩 속도가 느리다**고 판단했습니다.

2. 두 번째는 네비게이터 UI의 경우 데스크탑 및 태블릿 환경에서는 특정 페이지 이동 이후 "숙소 찾기" 버튼이 검색 페이지로 이동하는 반면, 모바일 환경에서는 메인 페이지로 이동하는 등 **디바이스에 따라 사용자 흐름이 일관되지 않는 문제**가 있었습니다.

3. 세 번째는 **좌우 슬라이드 영역이 의도와 다르게 상하 스크롤로 동작**하고, **카드 UI의 Scale-up 애니메이션이 상위 요소 영역을 벗어나면서 UI가 잘리는 현상**이 발생했습니다. 또한 뷰포트 너비가 약 768px 이하일 경우 탭 네비게이션 UI가 하단에 고정되지만, 해당 CSS 선택자에 **`max-width: 440px`이 적용되어 전체 너비가 맞지 않는 문제**도 확인했습니다.

<br />

이처럼 **기술적 난이도가 높은 "결제 시스템", "채팅 시스템" 등을 구현**하고, **실제 서비스에서 발생할 수 있는 다양한 문제를 마주하며 이를 개선하는 과정**을 실제 서비스를 운영하지 않고도 경험할 수 있다고 판단하여 해당 프로젝트를 진행하게 되었습니다.
